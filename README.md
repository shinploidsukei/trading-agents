Distributed Event-Driven Trading Agents System

A distributed trading simulation platform built with Python, Kafka, and Docker that demonstrates how multiple trading agents can process real-time market data in parallel. The system follows an event-driven architecture where market events are streamed through Kafka and consumed by distributed trading agents that generate trading decisions. A coordination and risk management service ensures consistent trade execution across the system.

This project was developed to explore distributed computing concepts, including parallel processing, message-driven communication, and fault tolerance in real-time systems.

Project Architecture

The platform consists of several microservices that communicate through Kafka event streams and run inside Docker containers.

Core Components

1. Market Data Producer

Generates simulated real-time market data (stock prices, trading events).

Publishes events to Kafka topics.

2. Kafka Message Broker

Serves as the event streaming platform.

Ensures reliable and ordered delivery of market events to all trading agents.

3. Distributed Trading Agents

Multiple agents run in parallel.

Each agent consumes market events from Kafka and executes its trading strategy.

Agents generate buy/sell signals based on incoming market data.

4. Coordination & Risk Management Service

Validates trading requests.

Prevents conflicting trades.

Maintains global account state and trade history.

5. Monitoring Dashboard

System metrics and trading activities are visualized using Grafana.

System Workflow

Market data is generated by the Market Data Producer.

Events are published to Kafka topics.

Trading agents subscribe to Kafka topics and process events.

Agents generate buy or sell signals.

Signals are validated by the Coordination & Risk Management Service.

Trade results and system metrics are monitored through Grafana dashboards.

Technologies Used

Python – Core application logic

Apache Kafka – Event streaming and messaging

Docker – Containerized microservices

Throughput – Number of events processed per second

Fault Tolerance – System behavior when containers fail
